Implement all student-facing modules in the existing CampusFlow project.

Use local JSON files or browser localStorage for data persistence.

Do not use MongoDB, JWT, external databases, or cloud storage.

==================================================

MODULE 1: SMART ISSUE REPORTING

Students can:

* Create Issue
* View Their Issues
* Track Status
* View Resolution Details
* Rate Resolution

Fields:

* Issue Title
* Description
* Location (Free Text)
* Category
* Priority
* Upload Photo

Create image upload functionality using local browser storage.

==================================================

AI ASSISTED CATEGORIZATION

When the student enters issue details:

Automatically suggest:

* Category
* Priority
* Responsible Department

Use keyword-based rule logic.

Examples:

water leakage → Maintenance → High Priority

broken fan → Electrical → Medium Priority

wifi problem → IT Support → Medium Priority

==================================================

ISSUE STATUS FLOW

Submitted
→ Under Review
→ Assigned
→ In Progress
→ Resolved

Students can see a timeline view.

==================================================

BEFORE / AFTER PROOF

When viewing resolved issues:

Display:

* Before Photo
* After Photo
* Resolution Notes

==================================================

RESOLUTION RATING

After issue becomes resolved:

Allow student to submit:

* 1 to 5 Stars
* Optional Comment

Store ratings and show them later in analytics.
